# UI Design Notes

## 设计目标

第三阶段的 UI 目标是让灵感银行更像一个个人思想资产操作系统，而不是营销页或炫技 demo。

关键词：

- 简洁
- 丝滑
- 克制
- 有资产感
- 轻金融感但不金融化
- 信息密度适中
- 可长期使用

## 页面层级

- Dashboard：展示整体资产状态、当前智能模式和推荐行动。
- 灵感列表：搜索、筛选、排序，卡片保持可扫描。
- 灵感详情：估值、分析、利息、内容输出分层展示。
- 新建灵感：低摩擦输入，明确“本地规则引擎”。
- 设置页：用户画像在前，智能引擎配置说明在后。
- 智能引擎评测中心：模型评测、zero-cost 指标、成本节省、失败样本。

## 色彩与卡片策略

- 背景使用浅灰绿和白色，避免强色块。
- 卡片统一 8px 圆角、轻边框、轻阴影。
- hover 只做轻微边框或位移，不做重动画。
- 数字清晰，但所有估值都标注“启发性估值”。
- 所有成本都标注“估算值”。

## 大厂风格参考

只学习方法，不照搬视觉：

- Linear：清晰层级和克制动效。
- Notion：轻量信息组织和空状态。
- Stripe Dashboard：专业的数据卡片。
- Apple 设置页：简洁留白和自然分组。
- Vercel Dashboard：开发者友好和低噪音。
- GitHub Projects：状态、标签、筛选。
- Raycast：命令感和低摩擦操作。

## 不做什么

- 不做营销型 landing page。
- 不使用夸张渐变和重阴影。
- 不把启发性估值包装成真实金融资产。
- 不使用夸大收益或金融化误导文案。
- 不让用户误以为必须配置外部 API。
- 不引入大型动画库或复杂设计系统。
